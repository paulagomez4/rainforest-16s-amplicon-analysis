================================================================================
BBDUK PARAMETERS
================================================================================

Input Format: Illumina paired-end FASTQ (gzip compressed)
Output Format: Quality-trimmed paired-end FASTQ (gzip compressed)

Quality Trimming:
  qtrim=rl              - Trim quality on both ends (right and left)
  trimq=10              - Trim to quality score Q10
  minlength=50          - Discard reads shorter than 50bp

Adapter/Contaminant Removal:
  ref=adapters,phix     - Remove Illumina adapters and PhiX
  ktrim=r               - Trim kmers to the right
  k=23                  - Kmer length
  mink=11               - Minimum kmer length
  hdist=1               - Hamming distance for kmer matching

Other:
  tpe                   - Trim both ends if adapter found
  tbo                   - Trim by overlap
  threads=10            - CPU cores to use

Expected Output: ~90-95% sequences retained

================================================================================
DADA2 DENOISING PARAMETERS (3-STEP SIMPLIFIED WORKFLOW)
================================================================================

Input: demux-paired-end.qza (quality-trimmed sequences from BBDuk)

Primer Removal (via trim-left parameters):
  Forward Primer (515F): GTGYCAGCMGCCGCGGTAA (19bp)
    trim-left-f: 19     - Remove from start of R1
    
  Reverse Primer (806wR): CCGYCAATTYMTTTRAGTTT (20bp)
    trim-left-r: 20     - Remove from start of R2

Rationale for direct trim-left:
  - Primers are uniform (from ACE sequencing facility)
  - No need for separate cutadapt step
  - Better preservation of sequence overlap for merging
  - Simpler workflow with higher reliability

Truncation Lengths:
  trunc-len-f: 250bp     - Truncate R1 to 250bp (quality drops after)
  trunc-len-r: 210bp     - Truncate R2 to 210bp (maintains higher quality)
  Rationale: Accounts for quality degradation, ensures sufficient overlap

Denoising Parameters:
  n-threads: 10          - CPU cores

DADA2 Workflow:
  1. Learn error models (per sample)
  2. Denoise (error correction)
  3. Merge paired-end reads
  4. Remove chimeras

Expected Output:
  - rep-seqs.qza: Representative ASV sequences
  - asv-table.qza: ASV abundance table
  - denoising-stats.qza: Denoising statistics

Expected Results (Rainforest Bark Project):
  - Total input reads: ~402,380
  - Merge rate: 82-90% ✅ (Good - sufficient overlap)
  - Non-chimeric rate: 78-90% ✅ (Good - reliable merging)
  - Final ASV count per sample: 50-500
  - Per-sample read depth: 13.5k - 66k reads

================================================================================
QUALITY THRESHOLDS
================================================================================

Minimum Quality (PHRED Q-scores):
  Q10: 90% accuracy
  Q20: 99% accuracy
  Q30: 99.9% accuracy

Forward reads: Start ~Q35, drop to Q20-25 by position 250
Reverse reads: Start ~Q35, maintain Q25-30 longer

This is typical for Illumina MiSeq 2x300bp runs.

Merge Rate Expectations:
  Good: >80% (indicates proper overlap)
  Fair: 70-80% (acceptable)
  Poor: <50% (indicates problems)

This project achieved: 82-90% ✅

Chimera Rate Expectations:
  Good: 10-25% removed as chimeric
  This project: 10-22% removed (excellent)

================================================================================
WORKFLOW EVOLUTION
================================================================================

Initial approach (NOT RECOMMENDED):
  - Separate cutadapt step with --p-discard-untrimmed
  - Result: Poor merge rates (30-70%), high false chimeras
  - Issue: Over-trimming reduced overlap between paired reads

Final approach (RECOMMENDED):
  - Direct trim-left in DADA2
  - Result: Excellent merge rates (82-90%), reliable data
  - Advantage: Simpler, more reliable, better results

Key lesson: For uniform primer sequences, DADA2's trim-left is preferable
to separate cutadapt step due to overlap preservation.
